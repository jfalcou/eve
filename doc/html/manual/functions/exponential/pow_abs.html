pow_abs¶

Required header: #include <eve/function/pow_abs.hpp>

namespace eve
{
   constexpr /* implementation defined */ pow_abs = {};
}

Function object computing \(|x|^y\).

Synopsis¶

template<typename T, typename U> auto operator()( T const& x, U const & y>
  • Computes element-wise absolute value of the first parameter raised to the power of the second.

Parameters¶

  • Each parameter x and y must be an instance of Value.
  • All Vectorized parameters must share the same type
  • If at least one parameter is Vectorized, all Vectorizable ones will be converted to its base type prior any other computation.
  • If all parameters are Vectorizable they must share the same Value type.

Return value¶

  • If any parameter is Vectorized, a value of this type else a value of the common type of the Vectorizable parameters.

Notes¶

  • Using pow_abs(x, y) for IEEEValue entries is similar to \(|x|^y\).

  • The implementation is conform to the IEEE requirements, but no exeception is ever raised.

    • pow_abs(\(\pm0\), exp), where exp is a negative odd integer, returns \(+\infty\)
    • pow_abs(\(\pm0\), exp), where exp is negative, finite, and is an even integer or a non-integer, returns \(+\infty\)
    • pow_abs(\(\pm0\), \(-\infty\)) returns \(+\infty\)
    • pow_abs(\(\pm0\), exp), where exp is a positive odd integer, returns \(+0\)
    • pow_abs(\(\pm0\), exp), where exp is positive non-integer or a positive even integer, returns \(+0\)
    • pow_abs(pm1, exp) returns 1 for any exp, even when exp is NaN
    • pow_abs(base, \(\pm0\)) returns 1 for any base, even when base is NaN
    • pow_abs(base, exp) returns NaN if base is finite and negative and exp is finite and non-integer.
    • pow_abs(base, \(-\infty\)) returns \(+\infty\) for any \(|\) base \(|<1\)
    • pow_abs(base, \(-\infty\)) returns \(+0\) for any \(|\) base \(|>1\)
    • pow_abs(base, \(+\infty\)) returns \(+0\) for any \(|\) base \(|<1\)
    • pow_abs(base, \(+\infty\)) returns \(+\infty\) for any \(|\) base \(|>1\)
    • pow_abs(\(\pm\infty\), exp) returns \(+0\) for any negative exp
    • pow_abs(\(\pm\infty\), exp) returns \(+\infty\) for any positive exp
    • except where specified above, if any argument is NaN, NaN is returned

  • With:ref:raw_ <feature-decorator> decorator uses the naive formula \(e^{y\log |x|}\) and so does not care for limits and leads to lower accuracy.

Example¶

#include <eve/function/pow_abs.hpp>
#include <eve/wide.hpp>
#include <eve/constant/inf.hpp>
#include <eve/constant/minf.hpp>
#include <eve/constant/nan.hpp>
#include <eve/constant/sqrtvalmax.hpp>
#include <iostream>

using wide_ft = eve::wide <float, eve::fixed<4>>;

int main()
{
  wide_ft pf = { 0.0f, 1.0f, -4.0f, -2.0f }; 
  wide_ft qf = { 1.0f, -1.0f, 3.0f, -0.0f};

  std::cout
    << "---- simd" << '\n'
    << "<- pf =                    " << pf << '\n'
    << "<- qf =                    " << qf << '\n'  
    << "-> eve::pow_abs(pf, qf) =  " << eve::pow_abs(pf, qf) << '\n'; 

  float xf = 2.0f;
  float yf = 10.0f;

  std::cout
    << "---- scalar"  << '\n'
    << "<- xf =                      " << xf << '\n'
    << "<- yf =                  " << yf << '\n'  
    << "-> eve::pow_abs(xf, yf) =    " << eve::pow_abs(xf, yf) << '\n'; 
  return 0;
}

Possible output:

---- simd
<- pf =                    (0, 1, -4, -2)
<- qf =                    (1, -1, 3, -0)
-> eve::pow_abs(pf, qf) =  (0, 1, 64, 1)
---- scalar
<- xf =                      2
<- yf =                  10
-> eve::pow_abs(xf, yf) =    1024
