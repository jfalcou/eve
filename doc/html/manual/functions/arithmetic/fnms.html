fnms¶

Required header: #include <eve/function/fnms.hpp>

namespace eve
{
   constexpr /* implementation defined */ fnms = {};
}

Function object performing a fused negate-multiply-sub between three Values of same element type

Synopsis¶

template<typename T,typename U,typename V>  auto operator()( T const& x, U cons & y, V const & z ) noexcept;
  • performs element-wise fused negate-multiply-sub of the three parameters

Parameters¶

  • Each parameter x, y and z must be an instance of a Value.
  • All Vectorized parameters must share the same type
  • If at least one parameter is Vectorized, all Vectorizable ones will be converted to its base type prior any other computation.
  • If all parameters are Vectorizable they must share the same Ieee Value type.

Return value¶

  • If any parameter is Vectorized, a value of this type else a value of the common type of the Vectorizable parameters.

Notes¶

  • The call fnms(x, y, z) is similar to -x*y-z

    But really conformant fused negate-multiply-sub also implies

    • only one rounding
    • nointermediate” overflow

    Our fnms provides this for all IntegralValue types and also each time it is reasonable in terms of performance for IEEEValue ones (i.e. if the system has the hard wired capability).

    If you need pedantic fnms capabilities in all circumstances in your own code you can use the pedantic_ or numeric_ decorators (although it can be very expensive).

  • With the pedantic_ decorator: ensures the fnms conformant properties and allows SIMD acceleration if available.

  • With the numeric_ decorator: decorator ensures the whole fma conformant properties and allows SIMD acceleration if available.

Example¶

#include <eve/function/fnms.hpp>
#include <eve/wide.hpp>
#include <iostream>

using wide_t = eve::wide<float, eve::fixed<4>>;

int main()
{
  wide_t oi = {2, -3, 0, 4};
  wide_t pi = {3, -2, -10, 0};
  wide_t qi = {4, -1, 1, 5};

  std::cout << "---- oimd" << '\n'
            << " <- oi =                          " << oi << '\n'
            << " <- pi =                          " << pi << '\n'
            << " <- qi =                          " << qi << '\n'
            << " -> eve::fnms(oi, pi, qi) =        " << eve::fnms(oi, pi, qi) << '\n';

  float wi = 3, xi = 3, yi = 4;
  std::cout << "---- scalar" << '\n'
            << " wi =                            " << wi << '\n'
            << " xi =                            " << xi << '\n'
            << " yi =                            " << yi << '\n'
            << " -> eve::fnms(wi, xi, yi) =       " << eve::fnms(wi, xi, yi) << '\n';
  return 0;
}

Possible output:

---- oimd
 <- oi =                          (2, -3, 0, 4)
 <- pi =                          (3, -2, -10, 0)
 <- qi =                          (4, -1, 1, 5)
 -> eve::fnms(oi, pi, qi) =        (-10, -5, -1, -5)
---- scalar
 wi =                            3
 xi =                            3
 yi =                            4
 -> eve::fnms(wi, xi, yi) =       -13
