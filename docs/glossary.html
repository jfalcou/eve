For any [scalar types](reference/concepts.html#scalar_value), the cardinal is equal to 1. For [SIMD types](reference/concepts.html#simd_value), it's equal to the number of lanes in the register. **EVE** provides the [cardinal](reference/types/traits.html#cardinal) traits to access the cardinal of any type. Element type ==================================================================================================== For any [value type](reference/concepts.html#value), **EVE** defines its underlying element type as the basic type used to represent its value. For any [scalar types](reference/concepts.html#scalar_value), its type is its own element type. For [SIMD types](reference/concepts.html#simd_value), it's defined as the scalar type associated. **EVE** provides the [element_type](reference/types/traits.html#value_type) traits to access the element type of any type. One say a type is from a scalar family if it satisfies the [scalar_value](reference/concepts.html#scalar_value) and from a SIMD family if it satisfies the [simd_value](reference/concepts.html#simd_value). Value type ==================================================================================================== For any [value type](reference/concepts.html#value), **EVE** defines its underlying value type as the basic type used to represent its value. For any [scalar types](reference/concepts.html#scalar_value), its type is its own value type. For [SIMD types](reference/concepts.html#simd_value), it's defined as the type of scalar value stored in the SIMD register. **EVE** provides the [value_type](reference/types/traits.html#value_type) traits to access the value type of any type. ----------------------------------------------------------------------------------------------------
