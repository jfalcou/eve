**Expressive Vector Engine** `eve::inc`
(insert ../../../crumbs.html here) Synopsis ==================================================================================================== **Required header:** Function object representing the inc unary operation. Member functions ==================================================================================================== Performs the [element-wise](../../../glossary.html#element-wise) increment by one operation on its parameter. Parameters ---------------------------------------------------------------------------------------------------- `x` : [value](../../concepts.html#value). Return value ---------------------------------------------------------------------------------------------------- Computes [element-wise](../../../glossary.html#elment-wise) a value with the same type as `x`. The result is `x+1`. Supported Incorators ==================================================================================================== [`saturated`](../incorators.html#saturated) : The call `saturated(inc)(x)` computes the saturated opposite of `x`. The only interest of this behaviour is that for integral type T `saturated(inc)(Valmax< T >())` returns `Valmax< T >()`. Masked calls ==================================================================================================== The [masked](../../../tutorial/masking.html) calls `inc[cond](x)` and `saturated(inc[cond])(x)` where `cond` satisfies the [conditionnal-expression](../../concepts.html#conditionnal) requirements are supported. More precisely, if `cond` is a mere [logical value](../../concepts.html#value) these expressions compute the increment (resp the saturated increment) of `x` if the condition `cond` is satisfied else returns `x`.
