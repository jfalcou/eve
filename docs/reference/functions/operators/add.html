**Expressive Vector Engine** `eve::add`
(insert ../../../crumbs.html here) Synopsis ==================================================================================================== **Required header:** Function object representing the add operation. Member functions ==================================================================================================== Performs the [element-wise](../../../glossary.html#regular) arithmetic sum of the parameter [values](../../concepts.html#value). Parameters ---------------------------------------------------------------------------------------------------- `x`, `args`, ... : [values](../../concepts.html#value). Return value ---------------------------------------------------------------------------------------------------- Return the sum of `x` and the `args`. The result type is the [compatibility result](../../concept.html#compatibility) of the parameters. The call `add(x, args, ...)` is equivalent to `(x + args + ...)` if `x` or one of the `args` is an [simd value](../../concepts.html#value). !!! WARNING Although the infix notation with `+` is supported, the `+` operator on standard scalar types is the original one and so can lead to automatic promotion. Supported decorators ==================================================================================================== [`saturated`](../decorator/saturated.html) : **Required header: ** : The call `saturated(add)(x, args...)` computes the saturated sum of `x` and the `args`. The saturation is obtained in the [compatibility result](../../concept.html#compatibility) of the two parameters. No overflow occurs. [`derivative`](../decorators.html#derivative) : **Required header: ** : The expression `derivative< N >(add)(x,args...)` computes the partial derivative of the function relative to its Nth parameter. The returned value is 1 if N is less or equal to the actual total parameter number, and 0 otherwise. !!! Warning This is only available for floating point entries. : Masked calls ==================================================================================================== The [masked](../../../tutorial/masking.html) calls `add[cond](x,y)` and `saturated(add[cond])(x,y)` where `cond` satisfies the [conditionnal-expression](../../concepts.html#conditionnal) requirements are supported. More precisely, if `cond` is a mere [logical value](../../concepts.html#value) these expressions compute the sum (resp the saturated sum) of `x` and `y` if the condition `cond` is satisfied else returns `x`. The calls are respectively equivalent to : * `if_else(cond, add(x,y),x)`, * `if_else(cond, saturated(add)(x,y),x)`; !!! Warning This is only available for two parameters.
