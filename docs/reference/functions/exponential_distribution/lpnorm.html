**Expressive Vector Engine** `eve::lpnorm`
(insert ../../../crumbs.html here) Synopsis ==================================================================================================== **Required header:** Function object representing the lpnorm operation. Member functions ==================================================================================================== Performs the [element-wise](../../../glossary.html#regular) lpnorm operation on its parameters. The value computed is the $l_2$ euclidian norm in $\mathbb{R}^n$ (where $n$ is the number of parameters : `sizeof...(args)+1`). Parameters ---------------------------------------------------------------------------------------------------- `p`, `x`, `args` : [floating value](../../concepts.html#value) Return value ---------------------------------------------------------------------------------------------------- For each [`element`](../../../glossary.html#element-wise) of the input: $\(x^p + args^p...)^{1/p}}$ is returned. The result type is the [compatibility result](../../concept.html#compatibility) of the parameters. Supported decorators ==================================================================================================== [`pedantic`](../decorators.html#pedantic) : **Required header: ** : The call `pedantic(lpnorm)(x,args...)` computes the $l_p$ norm without undue overflow or underflow at intermediate stages of the computation and can be more accurate than the [`regular`](../decorators.html#regular) call. Morever it returns $\infty$ as soon as one of its parameter is infinite, regardless of possible `Nan` values. [`diff`](../decorators.html#diff) : **Required header: ** : The expression `diff_< N >(lpnorm)(x,args...)` computes the partial diff of the function relative to its Nth parameter. The returned value is 0 if N is greater that the actual number of parameters, otherwise it is the `N`th parameter value divided by the `lpnorm` value.
