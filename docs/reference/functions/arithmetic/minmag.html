**Expressive Vector Engine** `eve::minmag`
(insert ../../../crumbs.html here) Synopsis ==================================================================================================== **Required header:** Function object representing the minmag operation. Member functions ==================================================================================================== Performs the minmag [element-wise](../../../glossary.html#regular) operator on its parameters. Parameters ---------------------------------------------------------------------------------------------------- `x`, `y` : [values](../../concepts.html#value) Return value ---------------------------------------------------------------------------------------------------- For each pair of corresponding [`elements`](../../../glossary.html#element-wise): * If `|x| > |y|`, `y` is returned. * If `|x| < |y|`, `x` is returned. * Otherwise `min(x, y)` is returned. The result type is the [compatibility result](../../concept.html#compatibility) of the two parameters. !!! WARNING If any element of the inputs is a `Nan`, the corresponding output element is system-dependent. supported Decorators ==================================================================================================== [`pedantic`](../decorators.html#pedantic) : The call `pedantic(minmag)(x,y)` is semantically equivalent to: * If `|x| > |y|`, `y` is returned. * If `|x| < |y|`, `x` is returned. * Otherwise `pedantic(min)(x, y)` is returned. [`numeric`](../decorators.html#numeric) : The call `numeric(minmag)(x,y)` ensures that if any element of the inputs is a `Nan`, the corresponding output element is the corresponding element of the other parameter (i.e. the not invalid parameter wins): * If `|x| > |y|`, `y` is returned. * If `|x| < |y|`, `x` is returned. * Otherwise `numeric(min)(x, y)` is returned.
