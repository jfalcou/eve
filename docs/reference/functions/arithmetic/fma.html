**Expressive Vector Engine** `eve::fma`
(insert ../../../crumbs.html here) Synopsis ==================================================================================================== **Required header:** Function object representing the fused multiply-add operation. Member functions ==================================================================================================== Performs the fused multiply-add operator on its parameters. Parameters ---------------------------------------------------------------------------------------------------- `x` : [value](../../concepts.html#value) `y` : [value](../../concepts.html#value) `z` : [value](../../concepts.html#value) Return value ---------------------------------------------------------------------------------------------------- - The result type is the [compatibility result](../../concept.html#compatibility) of the three parameters. - The call ``fma(x, y, z)`` is similar to ``x*y+z`` But really conformant fused multiply-add also implies - only one rounding - no "intermediate" overflow Our ``fma`` provides this for all [integral value](../../concepts.html#value) types and also each time it is reasonable in terms of performance for [real floating value](../../concepts.html#value) ones (i.e. if the system has the hard wired capability). If you need conformant fma capabilities in all circumstances in your own code you can use the [pedantic_](../../decorators.html#pedantic_) or [numeric_](../../decorators.html#numeric_) decorators (although this can be very expensive). Supported Decorators ==================================================================================================== [`pedantic_`](../decorators.html#pedantic_) : The call `pedantic_(fma)(x,y,z)` ensures the one rounding property. [`numeric_`](../decorators.html#numeric_) : The call `pedantic_(fma)(x,y,z)` ensures the whole fma conformant properties.
