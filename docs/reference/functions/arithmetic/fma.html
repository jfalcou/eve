**Expressive Vector Engine** `eve::fma`
(insert ../../../crumbs.html here) Synopsis ==================================================================================================== **Required header:** Function object representing the fused multiply-add operation. Member functions ==================================================================================================== Performs the [element-wise](../../../glossary.html#regular) fused multiply-add operation on its parameters. Parameters ---------------------------------------------------------------------------------------------------- `x`, `y`, `z` : [values](../../concepts.html#value) Return value ---------------------------------------------------------------------------------------------------- The call `fma(x, y, z)` is similar to `x*y+z` as if calculated to infinite precision and rounded once to fit the result as much as supported by the hardware. The result type is the [compatibility result](../../concept.html#compatibility) of the three parameters. !!! WARNING Note This `fma` implementation provides those properties for all [real integral values](../../concepts.html#integral_value) and when possible for [real floating values](../../concepts.html#value). Supported Decorators ==================================================================================================== [`pedantic`](../decorators.html#pedantic) : The call `pedantic(fma)(x,y,z)` ensures the one rounding property. This can be very expensive if the system has no hardware capability. [`numeric`](../decorators.html#numeric) : The call `numeric(fma)(x,y,z)` ensures the full compliance to fma properties. This can be very expensive if the system has no hardware capability.
