**Expressive Vector Engine** `eve::max`
(insert ../../../crumbs.html here) Synopsis ==================================================================================================== **Required header:** Function object representing the max operation. Member functions ==================================================================================================== Performs the [element-wise](../../../glossary.html#regular) max operation on its parameters. Parameters ---------------------------------------------------------------------------------------------------- `x`, `y` : [values](../../concepts.html#value) Return value ---------------------------------------------------------------------------------------------------- For each pair of corresponding [`elements`](../../../glossary.html#element-wise), the greater is returned. The result type is the [compatibility result](../../concept.html#compatibility) of the two parameters. !!! WARNING If any element of the inputs is a `Nan`, the corresponding output element is system-dependent. supported Decorators ==================================================================================================== [`pedantic`](../decorators.html#pedantic) : The call `pedantic(max)(x,y)` ensures the conformity to the standard behaviour, that is (on an [element-wise](../../../glossary.html#value) basis) semanticaly equivalent to: `(x < y) ? y : x`. [`numeric`](../decorators.html#numeric) : The call `numeric(max)(x,y)` ensures that if any element of the inputs is a `Nan`, the corresponding output element is the corresponding element of the other parameter (i.e. the not invalid parameter wins) (insert ../../src/max.src.html here) (insert ../../out/max.out.html here) See Also ==================================================================================================== [eve::min](./min.html), [eve::maxmag](./maxmag.html), [eve::minmag](./minmag.html).
