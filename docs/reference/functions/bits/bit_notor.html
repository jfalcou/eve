**Expressive Vector Engine** `eve::bit_notor`
(insert ../../../crumbs.html here) Synopsis ==================================================================================================== **Required header:** Function object representing the bitwise NOTOR operation. Member functions ==================================================================================================== Performs the [element-wise](../../../glossary.html#regular) bitwise NOTOR of the two [values](../../concepts.html#value). Parameters ---------------------------------------------------------------------------------------------------- `x`, `y` : [values](../../concepts.html#value). Return value ---------------------------------------------------------------------------------------------------- Computes bitwise NOTOR of the two parameters following the [bitwise operations semantic](../../../glossary.html#Bitwise). the call `bit_notor(x, y)` is semantically equivalent to `~x | y`. !!! WARNING Although the infix notation with `|` and `~` is supported for all [simd values](../../concepts.html#value), the `|` operator on standard scalar type is the original one and so does not allow floating parameters. Moreover if a proper intrinsic exists for NOTOR the infix notation will not be able to use it.
